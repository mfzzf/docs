---
title: "Authentication"
description: "Authenticate public Platform API calls and internal runtime calls."
---

Public Platform API calls use workspace API keys:

```bash
Authorization: Bearer sk-af-...
Content-Type: application/json
```

Generate and revoke API keys from the Astraflow workspace settings page. A key
is scoped to the workspace it was created in, so requests such as listing
agents, starting sessions, and reading usage do not need a workspace id in the
path.

Runtime endpoints are internal service-to-service endpoints called by the Web
backend. They use a separate shared runtime token:

```bash
X-Runtime-Token: $RUNTIME_INTERNAL_TOKEN
Content-Type: application/json
```

<Warning>
  Do not expose either token in frontend code. Platform API keys belong in your
  server, worker, or CI environment. Runtime tokens should only be shared
  between Astraflow services.
</Warning>

---
title: "Quickstart"
description: "Create an agent, start a session, and stream its events."
---

This guide starts from the public Platform API. You will list available models,
create an agent, start a session, and stream events until the run finishes.

Set the API base and a workspace API key:

```bash
export ASTRAFLOW_API_KEY="sk-af-..."
export ASTRAFLOW_BASE_URL="https://agents.plumzz.com/api"
```

<Warning>
  Keep workspace API keys on the server side. Do not embed them in browser
  clients or mobile apps.
</Warning>

<Steps>
  <Step title="Choose a model">
    Models are managed by ModelVerse and exposed through Astraflow as selectable
    agent models.

    ```bash
    curl "$ASTRAFLOW_BASE_URL/v1/models?limit=20" \
      -H "Authorization: Bearer $ASTRAFLOW_API_KEY"
    ```
  </Step>

  <Step title="Create an agent">
    An agent stores the instruction, model, tool policy, attached skills, MCP
    server references, memory stores, and runtime defaults. The exact payload
    can be expanded in the API Reference; this is the minimal shape for a first
    agent.

    ```bash
    curl -X POST "$ASTRAFLOW_BASE_URL/v1/agents" \
      -H "Authorization: Bearer $ASTRAFLOW_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Research assistant",
        "description": "Reads files and summarizes findings.",
        "instructions": "Be precise, cite file names, and ask before destructive changes.",
        "model": "gpt-5.5"
      }'
    ```
  </Step>

  <Step title="Start a session">
    Sessions are individual runs of an agent. Each session has its own prompt,
    files, approval mode, event stream, and final status.

    ```bash
    curl -X POST "$ASTRAFLOW_BASE_URL/v1/sessions" \
      -H "Authorization: Bearer $ASTRAFLOW_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "agent_id": "agt_...",
        "message": "List the files in the workspace and summarize what this project does.",
        "approval_mode": "auto_sandbox"
      }'
    ```
  </Step>

  <Step title="Stream events">
    Use server-sent events to render the session timeline. Events include user
    messages, assistant messages, tool calls, approval requests, usage, and
    terminal states.

    ```bash
    curl -N "$ASTRAFLOW_BASE_URL/v1/sessions/sess_.../stream" \
      -H "Authorization: Bearer $ASTRAFLOW_API_KEY"
    ```
  </Step>
</Steps>

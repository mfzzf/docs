---
title: "Quickstart"
description: "Run a managed-agent session and stream its events."
---

The normal product flow starts in the Web app: a workspace member creates an
agent, selects an environment, and starts a session. The Web backend then calls
the runtime with the internal token.

<Steps>
  <Step title="Create a session in the Web app">
    Create the `sessions` row with an active `agent_id`, `agent_version_id`,
    optional `environment_id`, and the user's first prompt in `input.prompt`.
  </Step>

  <Step title="Start the runtime turn">
    The Web backend starts the first turn with:

    ```bash
    curl -X POST "$RUNTIME_URL/internal/sessions/sess_xxx/runs" \
      -H "X-Runtime-Token: $RUNTIME_INTERNAL_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"message":"List the workspace files and summarize them."}'
    ```
  </Step>

  <Step title="Stream events">
    Subscribe to server-sent events to update the UI as messages, tool calls,
    approvals, and usage arrive.

    ```bash
    curl -N "$RUNTIME_URL/internal/sessions/sess_xxx/events?after_seq=0" \
      -H "X-Runtime-Token: $RUNTIME_INTERNAL_TOKEN"
    ```
  </Step>

  <Step title="Resume after approval">
    If a tool call requires human approval, submit the decision with the pending
    `tool_call_id`.

    ```bash
    curl -X POST "$RUNTIME_URL/internal/sessions/sess_xxx/approvals" \
      -H "X-Runtime-Token: $RUNTIME_INTERNAL_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"tool_call_id":"tool_xxx","decision":"approve"}'
    ```
  </Step>
</Steps>
